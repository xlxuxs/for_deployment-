Purpose

This file gives concise, actionable guidance for AI coding agents working in this repository. Follow the "link, don't embed" principle: reference existing docs rather than copying them.

Quick local run (developer)
---------------------------
- Backend: 
  - cd backend
  - npm install
  - cp .env.example .env (edit values)
  - npm run dev
- Frontend: 
  - cd frontend
  - npm install
  - npm run dev
- AI service (optional/local):
  - cd ai-service
  - pip install -r requirements.txt
  - python -m uvicorn app.main:app --reload

Deployed
--------

Conventions & notes for agents
------------------------------
- Dual AI modes: the AI service supports remote (Hugging Face/remote) and local model modes. Prefer linking to `ai-service/API_DOCS.md` for specifics.
- Policy lifecycle: policies have automated scheduled transitions (cron/worker). Check `backend/workers` for automation logic.
- Rate-limiting: Redis-backed rate limiters are used. See `src/config/rateLimits.js` and Redis config in `backend/.env.example`.
- SMS simulation and admin scripts live under `backend/scripts`.

How to behave
